# Daily TG Bot

Бот раз в день отправляет двум пользователям одно сообщение из заранее
заданного пула. Сообщения выбираются случайно, без повторов, пока весь пул
не будет пройден — после этого пул перемешивается заново.

Время отправки: **20:00 по Москве** (17:00 UTC).

## Что нужно сделать перед первым запуском

### 1. Заполнить пул сообщений
Открой `messages.json` и впиши свои тексты вместо примеров.
Можно добавлять/удалять строки — главное сохранять валидный JSON-массив строк.

### 2. Узнать токен бота
У `@BotFather` в Telegram → `/newbot` → получишь токен вида
`123456:ABC-DEF...`.

### 3. Узнать chat_id двух получателей
Каждый получатель пишет `@userinfobot` в Telegram — бот присылает его
числовой ID. Также оба получателя должны написать `/start` твоему боту
(иначе бот не сможет им писать).

### 4. Добавить секреты в репозиторий
В репозитории на GitHub: **Settings → Secrets and variables → Actions →
New repository secret**. Добавить три секрета:

| Имя секрета | Значение |
|---|---|
| `BOT_TOKEN` | токен от BotFather |
| `CHAT_ID_1` | chat_id первого получателя |
| `CHAT_ID_2` | chat_id второго получателя |

### 5. Загрузить файлы в репозиторий
Структура должна быть такой:
```
.github/workflows/daily.yml
messages.json
state.json
send.py
```

### 6. Проверить вручную
Вкладка **Actions** в репозитории → выбрать workflow "Daily message" →
**Run workflow**. Это запустит отправку немедленно, не дожидаясь
расписания — удобно для проверки, что всё работает.

## Как это работает

- `state.json` хранит список индексов сообщений, которые ещё не были
  отправлены в текущем "круге". Когда список пустеет — генерируется
  заново и перемешивается.
- После каждой отправки workflow коммитит обновлённый `state.json`
  обратно в репозиторий — так состояние не теряется между запусками
  (GitHub Actions не имеет постоянного диска, поэтому это и есть
  "облачное хранение" для бота).
- `cron: "0 17 * * *"` означает 17:00 UTC каждый день. Если нужно
  поменять время отправки — поменяй эту строку в `daily.yml`
  (не забывай про разницу UTC/МСК = +3 часа).

## Важно знать про GitHub Actions cron

GitHub не гарантирует запуск ровно по расписанию — возможна задержка
до 5-15 минут в часы высокой нагрузки на серверах GitHub. Для
ежедневного напоминания это обычно не критично.
